# 车间传感器质检分析工具

一款纯前端的工业传感器时间序列质检分析工具，支持规则配置、CSV/JSON 导入、导入预检、异常检测、人工复核、版本回滚和报告导出。所有数据保存在浏览器本地 IndexedDB 中，无需后端服务。

## 功能特性

### 1. 规则配置
- **设备管理**：配置车间生产线设备
- **传感器范围**：设置每个传感器的最小/最大值、跳变阈值、单位
- **缺失规则**：配置时间最大间隔和连续缺失点数阈值
- **版本管理**：规则修改保存为新版本，可随时切换或回滚

### 2. 数据导入预检
- **CSV/JSON 双入口**：拖拽或点击上传 CSV 或 JSON 文件
- **导入预检**：在真正导入前，先展示字段映射、时间解析结果、冲突提示、异常摘要和关键列预览
- **字段映射检测**：自动识别 timestamp/time/ts、sensor/sensorName、value/val/reading 等字段别名
- **时间解析预览**：显示格式分布、解析冲突、受影响行号和最终采用规则
- **异常摘要**：预检阶段即可看到预计检测到的异常数量和类型分布
- **关键列预览**：展示前 10 行最终写入数据库的数据（原始时间、标准化时间、传感器、数值、采用格式）
- **CLI 预览命令**：`npm run cli-preview <file>` 在命令行中预览导入结果
- 实时校验：坏时间戳、未知传感器、非法数值
- 批次号查重，防止重复导入
- 一键生成内置 CSV/JSON 样例数据

### 3. 时间配置按数据源记忆
- 时间格式配置不再仅全局保存，而是**按数据源（文件名）记住上次选择**
- 支持自动识别、秒级 Unix、毫秒 Unix、自定义格式
- 用户改过后重启再回来也能恢复
- 自动判断和手动配置冲突时，明确说明冲突原因、受影响行、最终采用规则
- 冲突决策写入可追溯日志，导出时包含原始时间、标准化时间、异常说明和采用规则

### 4. 异常检测
自动检测 4 类时间序列异常：
| 异常类型 | 说明 |
|---------|------|
| 数据缺失 | 时间间隔超过配置阈值或连续缺失点 |
| 越界异常 | 传感器数值超出配置的 [min, max] 范围 |
| 跳变异常 | 相邻数值差超过跳变阈值 |
| 重复时间戳 | 同一传感器出现相同时间戳 |

### 5. 人工复核
- 单条打标：确认故障、误报、忽略
- 批量打标：勾选多条异常一键标记
- 上下文图表：折线图展示异常点前后 20 个数据点
- 实时统计摘要：标签变化时统计数据自动更新

### 6. 版本回滚
- 批次可回滚到任意历史规则版本
- 记录完整回滚痕迹（时间、原因、源版本、目标版本）
- **不会清除已有的人工复核标注**

### 7. 历史记录 & 报告导出
- 所有批次搜索、状态追踪
- 报告 CSV 导出：包含批次信息、统计摘要、异常清单、复核结果
- 重启后数据保持一致：规则版本、异常编号、复核标签、回滚痕迹、统计摘要全部持久化

## 技术栈

- **框架**：React 18 + TypeScript 5 + Vite 6
- **样式**：Tailwind CSS 3（工业风配色）

- **本地数据库**：IndexedDB (Dexie.js 4)
- **图表**：Recharts 2

## 数据格式

### CSV 格式

CSV 文件必须包含以下 3 列，顺序可以任意：

```csv
timestamp,sensorName,value
2024-01-15 08:00:00,Temperature_A1,65.3
2024-01-15 08:01:00,Temperature_A1,66.1
2024-01-15 08:02:00,Pressure_B2,2.35
```

### JSON 格式

JSON 文件支持以下 3 种结构：

**1. 对象数组**
```json
[
  { "timestamp": "2024-01-15 08:00:00", "sensorName": "Temperature_A1", "value": 65.3 },
  { "timestamp": "2024-01-15 08:01:00", "sensorName": "Temperature_A1", "value": 66.1 }
]
```

**2. data 字段包裹**
```json
{
  "data": [
    { "timestamp": "2024-01-15 08:00:00", "sensorName": "Temperature_A1", "value": 65.3 }
  ]
}
```

**3. records 字段包裹**
```json
{
  "records": [
    { "timestamp": "2024-01-15 08:00:00", "sensorName": "Temperature_A1", "value": 65.3 }
  ]
}
```

字段别名支持：`timestamp`/`time`/`ts`、`sensorName`/`sensor`、`value`/`val`/`reading`

### 支持的时间戳格式
- `2024-01-15 08:00:00`
- `2024-01-15T08:00:00Z`
- Unix 时间戳（毫秒）

## 样例复核步骤

以下是完整的端到端操作流程示例：

### 步骤 1：配置质检规则

1. 打开左侧导航栏「**规则配置**」页面
2. 查看默认规则 v1，已预置 3 个传感器：
   - Temperature_A1（温度，范围 40~100℃，跳变阈值 30）
   - Pressure_B2（压力，范围 1.0~5.0MPa，跳变阈值 1.5）
   - Vibration_C3（振动，范围 0.1~2.0mm/s，跳变阈值 0.8）
3. 可按需修改传感器范围或新增传感器
4. 修改后点击「**保存为新版本**」，输入规则名称和描述，生成 v2 版本

### 步骤 2：导入样例数据

1. 打开左侧导航栏「**数据导入**」页面
2. 选择文件格式：**CSV** 或 **JSON**
3. 点击「**下载标准样例**」按钮下载预置的测试数据（包含 4 类异常样本）
4. 也可拖拽自己的 CSV 或 JSON 文件到上传区域
5. 系统自动进行**导入预检**，展示：
   - **字段映射**：自动识别字段名和别名
   - **时间解析预览**：格式分布、冲突数、解析错误数
   - **异常摘要**：预计检测到的异常数量和类型
   - **关键列预览**：前 10 行最终写入的数据
6. 配置时间格式（自动识别 / Unix 秒 / Unix 毫秒 / 自定义）
   - 配置按数据源自动记住，下次打开同一文件会恢复上次选择
7. 填写批次号（自动生成）和备注，确认无重复批次
8. 点击「**开始检测并导入**」，系统自动导入数据并运行异常检测

### 步骤 3：异常复核

1. 检测完成后自动跳转到「**异常复核**」页面
2. 顶部展示统计摘要卡片：异常总数、确认故障数、误报数、忽略数、完成率
3. 中间图表区：异常类型饼图、复核标签饼图、传感器分布柱状图
4. 下方异常列表：
   - 使用「**筛选器**」按异常类型或复核标签过滤
   - 点击行左侧 **▼** 展开异常详情和上下文折线图
   - 勾选多行可批量操作
5. 单条打标（每行右侧三个按钮）：
   - ✅ **确认故障**：红色，表示真实异常
   - ✕ **误报**：绿色，表示算法误报
   - − **忽略**：灰色，表示无需关注
6. 批量打标：勾选多行后，点击列表上方的批量按钮
7. 统计摘要和图表会**实时更新**，反映最新的复核进度

### 步骤 4：版本回滚（可选）

1. 打开「**历史记录**」页面
2. 找到目标批次，点击「**回滚**」按钮
3. 在弹出框中选择目标规则版本（必须是不同版本）
4. 填写回滚原因，例如：「之前阈值配置过严」
5. 可选择是否用新规则重新检测异常
6. 回滚后批次状态变为「已回滚」，回滚痕迹显示在历史列表中

### 步骤 5：导出报告

1. 在「异常复核」页面点击右上角「**导出报告**」按钮
2. 或在「历史记录」中点击「**查看报告**」
3. 报告页面展示：
   - 批次信息（编号、导入时间、数据量、规则版本）
   - 统计摘要卡片
   - 可视化图表
   - 完整异常清单
4. 点击「**下载 CSV 报告**」导出到本地，文件包含：
   - 批次元信息
   - 统计摘要
   - 按类型/传感器分布
   - 每条异常的详情和复核结果

### 数据一致性验证

- 关闭浏览器再重新打开，进入「历史记录」
- 点击同一批次查看：
  - 规则版本号不变
  - 异常 ID 编号不变
  - 所有复核标签保持原样
  - 回滚痕迹完整保留
  - 统计摘要完全一致

## 错误反馈说明

| 场景 | 反馈方式 |
|-----|---------|
| 坏时间戳 | 导入页面红色错误表格，显示行号、列、错误原因 |

## 默认规则说明

首次打开系统会自动创建默认规则 v1：

| 传感器名称 | 范围 | 跳变阈值 | 单位 |
|-----------|-----|---------|-----|
| Temperature_A1 | 40 ~ 100 | 30 | ℃ |
| Pressure_B2 | 1.0 ~ 5.0 | 1.5 | MPa |
| Vibration_C3 | 0.1 ~ 2.0 | 0.8 | mm/s |

所有传感器默认缺失规则：最大间隔 120 秒，连续缺失 2 点即报缺失异常。

## 标签历史可追溯机制

本工具实现了完整的复核标签变更历史追溯，确保审计可追踪：

### 存储设计
- **追加式存储**：每次修改标签时，不会覆盖旧记录，而是追加新的决策记录
- **旧记录标记**：历史记录标记 `isSuperseded: true`，最新记录标记 `isSuperseded: false`
- **版本序号**：`sequence` 字段自动递增，记录变更顺序
- **前标签记录**：`previousLabel` 字段记录变更前的标签，形成完整变更链

### 数据一致性保证
1. **统计口径**：`computeStatistics` 会自动从历史中提取最新有效决策（通过 `collapseDecisions` 函数）
2. **重启恢复**：从 IndexedDB 加载所有历史记录后，按时间线折叠出最新状态，确保重启前后统计完全一致
3. **报告导出**：CSV 报告新增「历史标签变更」列，完整展示 `1. 确认故障 → 2. 误报 → 3. 忽略` 等变更链
4. **永不删除**：所有历史决策永久保留，支持审计追溯

### 测试验证
运行 `npm run test:integration` 可验证以下场景：
- 确认故障 → 改判误报 → 改判忽略，历史完整保留
- 模拟浏览器重启（关闭/打开 IndexedDB 连接），数据完全一致
- 重启前后统计摘要（故障数、误报数、完成率等）数字完全相同
- 导出报告包含完整历史标签变更痕迹

## 导入预检架构

### 模块分层

```
数据源 (CSV/JSON)
  ↓
jsonParser / csvParser  →  字段映射 + 行解析 + 时间解析
  ↓
importPrecheck          →  预检流水线 (映射→时间预览→异常摘要→关键列)
  ↓
┌──────────┬──────────┐
│ GUI      │ CLI      │
│ Import.tsx│ cli-preview.ts │
└──────────┴──────────┘
```

### 关键设计

- **确定性 ID**：DataPoint ID 使用 `dp_{sourceHash}_{rowNum}` 格式，同一文件重复导入生成相同 ID，支持撤销后重新导入得到同样结果
- **按数据源时间配置**：每个数据源（文件名）独立保存时间格式选择，通过 localStorage 持久化，重启后自动恢复
- **冲突透明**：自动识别与手动配置冲突时，记录冲突行号、原始值、自动格式、手动格式、采用规则、原因
- **预检结果**：`PrecheckResult` 包含字段映射、时间解析预览、异常摘要、关键列预览、校验结果

### CLI 使用

```bash
# 基本用法
npm run cli-preview data.csv
npm run cli-preview data.json --format json

# 指定时间格式
npm run cli-preview data.csv --time-preset unix_seconds
npm run cli-preview data.csv --time-preset custom --custom-format "YYYY-MM-DD HH:mm:ss"
```
